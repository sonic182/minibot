Invoke a specialist agent to handle a delegated task.

Use this when the task is better handled by a domain specialist (for example, complex multi-step work or tasks tied to a known specialist capability).

If you are not certain which specialist to use, call list_agents first.

Provide:
- agent_name: exact name from list_agents.
- task: clear, concrete objective.
- context: optional supporting details that improve execution quality.
- mode: optional `async` or `sync`. Omit it to use `async`.

Use `async` for long-running work. It queues a background job and returns immediately with a job id, so you should tell the user the task was started and results will arrive later. If async jobs are not enabled in the runtime, the tool returns an explicit error instead of silently switching modes.

Use `sync` only for short work that must finish in the same turn. In that mode, wait for the delegated result and then continue with your own final response to the user.
